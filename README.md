🃏 Daifugo Card Game (대부호 게임)
📌 프로젝트 소개

이 프로젝트는 Unity로 구현한 싱글 플레이 대부호(다이후고) 카드 게임입니다.
플레이어 1명과 AI 4명이 함께 플레이하며, 카드 조합과 다양한 특수 효과를 활용하여 손패를 먼저 소진하는 것이 목표입니다.

단순한 카드 제출 게임이 아니라,
라운드 → 계급 결정 → 다음 게임에 영향으로 이어지는 구조를 가진 전략 카드 게임입니다.

🎮 핵심 게임 플레이

카드 선택 및 제출
→ 조합 판정
→ 특수 효과 발생
→ 턴 진행 / 라운드 종료
→ 손패 소진 → 순위 확정
→ 계급 결정
→ 다음 게임에 영향

⭐ 주요 특징

1. 계급 시스템 (Hierarchy System)

Daifugo / Fugo / Heimin / Hinmin / Daihinmin
게임 결과에 따라 계급이 결정됨
다음 게임 시작 시 카드 교환 발생

2. 다양한 특수 카드 효과
   
5 : 다음 플레이어 스킵
7 : 카드 전달 (Give)
8 : 라운드 즉시 종료
9 : 턴 방향 반전
10 : 카드 버리기
J : 11 Back (서열 반전)
4장 이상 또는 스트레이트 : 혁명

3. Tight 시스템 (고정 규칙)

숫자 고정 (Number Tight)
문양 고정 (Suit Tight)

라운드 도중 특정 조건을 만족하면 이후 카드 제출 조건이 제한됩니다.

4. 복합 효과 처리

한 번의 카드 제출로 여러 효과가 동시에 발생할 수 있으며,
효과는 정해진 순서에 따라 처리됩니다.

🧠 시스템 구조

GameManager
게임 전체 흐름 제어
턴 진행, 라운드 시작/종료, UI 갱신
RuleManager
카드 조합 판정
제출 가능 여부 판단
게임 규칙 중앙 관리
EffectProcessor
특수 효과 적용
상태 변화 처리 (혁명, 반전, 스킵 등)
CardSubmissionService
카드 제출 처리
테이블 상태 업데이트
RoundService
패스 처리
라운드 종료 판정
ResultService
순위 및 계급 결정
게임 종료 조건 판단
FollowUpSelectionController
7, 10 효과 처리
추가 입력 기반 흐름 관리
RoundConstraintService
Tight 규칙 관리
제출 이후 제한 조건 갱신
HierarchyExchangeController
계급 교환 처리
카드 교환 로직 관리
PresentationController
메시지 및 연출 출력
이벤트를 순서대로 처리하는 구조 (Queue 사용)

📦 사용된 자료구조
Array : 플레이어 목록 관리
List : 카드 목록, 이벤트 리스트
Dictionary : 데이터 매핑
Queue : 연출 이벤트 순차 처리

🛠️ 기술 요소
Unity (C#)
Coroutine 기반 비동기 처리
상태 기반 게임 흐름 관리
이벤트 기반 연출 시스템
🔧 주요 구현 포인트
특수 효과 간 충돌을 방지하기 위한 고정된 처리 순서 설계
입력이 필요한 효과와 즉시 효과를 분리한 구조
게임 로직과 UI/연출의 분리
유지보수를 고려한 서비스 단위 구조
